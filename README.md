## Overview

Runtime Stability is a technology framework 
for structurally maintaining seven Protection 
Attributes when a computer system faces 
cyberattacks, failures, or anomalies during 
execution (runtime).

### 7 Protection Attributes

| Abbr | Attribute | Description |
|------|-----------|-------------|
| S | Safety | System remains safe even under attack |
| R | Reliability | Protection functions operate continuously |
| A | Availability | Service delivery is maintained |
| C | Controllability | System state can be guided to safe state |
| Cf | Confidentiality | In-process memory confidentiality maintained |
| D | Data Integrity | Accuracy and completeness of data maintained |
| Ix | Inexploitability | Vulnerabilities structurally absent at runtime |

### Evaluation Scale
```
RS = SL + IL

SL: Runtime Security Level (0–3)
IL: Runtime Immunity Level (0–3)
RS: Runtime Stability Level (0–6)
```

| RS Level | Application Domain |
|----------|-------------------|
| RS-2 | General web applications |
| RS-3 | SaaS, online trading |
| RS-4 | Financial institutions, healthcare |
| RS-5 | Autonomous vehicles |
| RS-6 | Defense, critical infrastructure |
